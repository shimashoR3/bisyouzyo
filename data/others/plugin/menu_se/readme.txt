【メニューSE追加プラグイン Ver2.01】
http://hororo.wp.xdomain.jp/198/

2021/07/30更新 v507b対応版

-------------------------------------------------------------------------------------------------
■機能
-------------------------------------------------------------------------------------------------
　●「メニューボタン（旧歯車アイコン）」「メニュー画面の各ボタン」「セーブ・ロード領域」
　 「メニュー、セーブ、ロード、バックログの BACK ボタン」「ダイアログのボタン」
　　にSE（効果音）を指定するプラグインです。

　●button・glink などにもSEを指定できます。（V2.00以降）
　　
　◆出来ない事
　　・メニューのボタン別の設定はできません。（※改造はOK）
　　・メニューに追加したボタンには対応できません。（※改造はOK）
　　・メニューのclass名を変更した場合は動作しません。
　　・本体改造をしてないので、若干微妙な所もあります。
　
　◆微妙な点
　　・公式標準の「タイトルに戻る」ダイアログの「OK」は、se再生を待たずにタイトルに戻るので音が鳴りません。
　　　（※環境や音声データによります）
　　・セーブ・ロードデータにenter時のSEを設定した場合、表示切替直後にenter時のSEが鳴る場合があります。


-------------------------------------------------------------------------------------------------
■導入方法
-------------------------------------------------------------------------------------------------
　① data/others/plugin に 「menu_se」フォルダを入れてください。
　
　② 使用している、save.html、load.html、menu.html、backlog.html をテキストエディタで開き、
　　一番下に下記コードを記入してください。

　　<script type="text/javascript" src="./data/others/plugin/menu_se/menu_se.js"></script>

　　※save.html、load.html、menu.html、backlog.html ファイルの場所はご自身でお確かめ下さい。

　③ ゲーム起動時に必ず読み込むファイル（first.ksなど）に下記コードを記述し、プラグインを呼び出してください。
　　[plugin name=menu_se]
　　
　④ パラメータで使用する効果音ファイルを指定してください。
　　[plugin name=menu_se se_clickse=***.ogg] など
　　※効果音のファイルはプロジェクトフォルダの sound フォルダに入れてください。

　⑤ [menu_se_set] タグでボタン別の指定もできます。詳しくは「記述例」を参照してください。

■パラメーター
-------------------------------------------------------------------------------------------------
　◆[plugin]タグ用（※不要な場合は省略可）
　　・共通（全てのボタンに同じSEを一括で指定できます）
　　　se_clickse    　　　　 //ボタンをクリックした時
　　　se_enterse    　　　　 //ボタンの上にマウスカーソルが乗った時
　　　se_leavese   　　　　  //ボタンの上からマウスカーソルが外れた時

　　・メニューボタン（歯車アイコン）
　　　button_menu_clickse   //ボタンをクリックした時
　　　button_menu_enterse   //ボタンの上にマウスカーソルが乗った時
　　　button_menu_leavese   //ボタンの上からマウスカーソルが外れた時

　　・メニュー画面のボタン
　　　menu_item_clickse     //ボタンをクリックした時
　　　menu_item_enterse     //ボタンの上にマウスカーソルが乗った時
　　　menu_item_leavese     //ボタンの上からマウスカーソルが外れた時

　　・メニュー/セーブ/ロード/バックログの閉じるボタン（共通）
　　　menu_close_clickse    //ボタンをクリックした時
　　　menu_close_enterse    //ボタンの上にマウスカーソルが乗った時
　　　menu_close_leavese    //ボタンの上からマウスカーソルが外れた時

　　・セーブデータ
　　　save_list_clickse     //ボタンをクリックした時
　　　save_list_enterse     //ボタンの上にマウスカーソルが乗った時
　　　save_list_leavese     //ボタンの上からマウスカーソルが外れた時

　　・セーブ/ロード/バックログのスマホ用スクロールボタン（共通）
　　　button_smart_clickse  //ボタンをクリックした時

　　・ダイアログのconfirm（OK）ボタン
　　　confirm_clickse       //ボタンをクリックした時（※タイトルバック時は鳴りません）
　　　confirm_enterse       //ボタンの上にマウスカーソルが乗った時
　　　confirm_leavese       //ボタンの上からマウスカーソルが外れた時

　　・ダイアログのcancelボタン
　　　cancel_clickse        //ボタンをクリックした時
　　　cancel_enterse        //ボタンの上にマウスカーソルが乗った時
　　　cancel_leavese        //ボタンの上からマウスカーソルが外れた時

　◆[menu_se_set]タグ用（※name以外は省略可）
　　　name                  //ボタンの名前（※必須）
　　　clickse               //ボタンをクリックした時
　　　enterse               //ボタンの上にマウスカーソルが乗った時
　　　leavese               //ボタンの上からマウスカーソルが外れた時

　　　nameの値（※[menu_se_set]では共通の指定はできません）
　　　　button_menu         //メニューボタン
　　　　menu_item           //メニュー画面のボタン
　　　　menu_close          //閉じるボタン
　　　　save_list           //セーブ・ロード
　　　　button_smart        //スマホ用スクロールボタン
　　　　confirm             //ダイアログのconfirm
　　　　cancel              //ダイアログのcancel
　　　　任意のname　　　　 　//button・glinkタグなどのnameを指定できます。
　　　　　　　　　　　　　　　　※button・glink 以外のタグの name に同じ名前を付けないよう注意してください。

■記述例
-------------------------------------------------------------------------------------------------

　◆[plugin]記述例
　　例：メニュー画面のボタンにSEを指定する場合。
　　　[plugin name=menu_se menu_item_clickse=click.ogg menu_item_enterse=enter.ogg menu_item_leavese=leave.ogg]

　　例：共通パラメータの使い方。
　　　[plugin name=menu_se se_clickse=click.ogg se_enterse=enter.ogg se_leavese=leave.ogg]
　　　※これで全てのボタンにSE指定が可能です。

　　　se_** よりも個別指定が優先されます。
　　　backボタンのクリック音のみ変える例です。
　　　[plugin name=menu_se se_clickse=click.ogg se_enterse=enter.ogg se_leavese=leave.ogg menu_close_clickse=close_clickse.ogg]
　　　
　　※パラメータが多くて分りにくい場合は、init.ks を直接編集する方が簡単です。


　◆[menu_se_set]使用例
　　例：メニュー画面のボタンとセーブデータにSEを指定する場合。
　　　[plugin name=menu_se]
　　　[menu_se_set name=menu_item clickse=click.ogg enterse=enter.ogg leavese=leave.ogg]
　　　[menu_se_set name=save_list clickse=click.ogg enterse=enter.ogg leavese=leave.ogg]

　　※[menu_se_set]は[plugin name=menu_se]の後に記述してください。
　　※[menu_se_set]で指定したSEが優先になります。
　　※シナリオ途中での変更も可能ですが、ロード時に対応させるにはmake.ksへの追記が必要です。


　　例：button タグ等に同じ name を指定する事で、纏めてSEを設定する事ができます。
　　　[menu_se_set name=se01 clickse=click.ogg enterse=enter.ogg leavese=leave.ogg]
　　　[button name=se01 role=save]
　　　[button name=se01 role=load]

　　　※button・glink など対象以外のタグの name に同じ名前を付けないよう注意してください。
